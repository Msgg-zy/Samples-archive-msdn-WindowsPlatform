Trial-mode scenario

This scenario illustrates how an app can determine what type of license the customer has and then enable the products that correspond to that license.

No license data.

Read the expiration date of the trial period, subtract today's date from the expiration date, and then display the result, converted to days, in the output area. After the customer buys the app, the expiration date is no longer meaningful.

You could use this to inform the customer that the trial period is ending soon and encourage them to buy the full version.

Read the current license state to enable a product during the trial period and display the result in the output area.

This might be used for a product that is only available before the customer buys the app, such as displaying a banner to indicate they are running a trial version.

Read the current license state to enable a product that can only be used after the customer buys the app. Display the result in the output area. Click Buy app to buy the app, and then try this button again.

Read the app's price from the Store API and display it here.

Click Buy app to display the purchase dialog box and update the license state according to the value it returns.
