To impose a set of security restrictions on content hosted by an iframe, use the sandbox attribute. When you specify the sandbox attribute on an iframe element, the content is said to be sandboxed. Sandboxed content can’t execute scripts, submit forms, or navigate the top level document. In addition, the content is forced into a unique origin that prevents it from accessing content from the same origin, such as cookies.

The sandbox attribute is useful for when you want to display untrusted content that doesn’t originate within your app. To learn about the different sandbox flags, click the examples in the Scenarios selector. (This SDK sample doesn’t cover the allow-top-navigation or ms-allow-popups flags, because you can’t perform top-level navigations to external content or show popups in a Windows Store app.)

These two iframe elements demonstrate the effects of sandboxing content. They both display the same example content, but one uses the sandbox attribute and one doesn’t.


<iframe>

Show <iframe> origin (location.host)


<iframe sandbox>
