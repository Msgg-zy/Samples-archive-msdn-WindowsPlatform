Locally packaged HTML rendered with the ms-appx:// URI scheme is known as the local context. HTML in this context has accidental script injection validation applied to functions that can accidentally inject dynamic content, such as innerHTML, outerHTML, and document.write. HTML in the local context has full access to Windows Runtime APIs to access connected devices, networking APIs, and other native Windows functionality.

HTML served via HTTP or HTTPS or locally packaged HTML rendered with ms-appx-web:// scheme is considered "Web context" and is treated just like normal web content in that it doesn’t have accidental script injection validation applied, and doesn’t get access to Windows Runtime APIs.
