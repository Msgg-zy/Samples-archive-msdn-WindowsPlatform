To allow a sandboxed page to access content from the same origin (such as cookies), use the sandbox attribute and specify the allow-same-origin flag. The sandboxed content is still prevented from executing scripts, submitting forms, and navigating the top-level document.

The allow-same-origin flag is useful for allowing content from the same site to be sandboxed to disable scripting, while still allowing access to the document of the sandboxed content. You can also use it to sandbox content from a 3rd party site while still allowing the sandboxed content to communicate with its originating site and access data associated with its origin, such as a cookies or IndexedDB.

If the iframe content has the same origin as the parent, don’t combine the allow-scripts and the allow-same-origin flags. If both tokens are used together and the origins match, the content within the iframe will be able to remove the sandbox attribute.

These two iframe elements demonstrate the effects of sandboxing content. They both display the same example content, but one uses the sandbox attribute with the allow-same-origin flag and one doesn’t.
